## Project Overview

Personal expense tracking app with a Java/Spring Boot REST API and an Angular frontend. All code and identifiers are in **English**.

## Commands

### Backend (`backend/`)

```bash
# Run in dev mode (requires postgres running)
mvn spring-boot:run

# Build jar
mvn package -DskipTests

# Run all tests
mvn test

# Run a single test class
mvn test -Dtest=TransactionServiceTest

# Run a single test method
mvn test -Dtest=TransactionServiceTest#shouldCreateTransaction

# Compile only (fast check)
mvn compile
```

### Frontend (`frontend/`)

```bash
# Dev server at http://localhost:4200
npm start

# Run unit tests (Karma/Jasmine)
npm test

# Build for production
npm run build

# Run a specific spec file
npx ng test --include='**/transaction.service.spec.ts'
```

### Docker

```bash
# Start only the database (for local backend dev)
docker compose up -d postgres

# Start full stack
docker compose up -d

# Run integration test database (port 5433)
docker compose -f docker-compose.test.yml up -d

# Tear down test database
docker compose -f docker-compose.test.yml down
```

## Architecture

### Backend

Layered architecture: `controller → service → repository → entity`

```
com.personalfinance/
├── config/          # Spring Security, JWT filter, CORS, beans
├── controller/      # REST endpoints, request mapping, auth
├── service/         # Business logic, transaction management
├── repository/      # Spring Data JPA interfaces
├── model/entity/    # JPA entities (User, Transaction, Category)
│   └── enums/       # TransactionType (INCOME | EXPENSE)
└── dto/
    ├── request/     # Inbound payloads with @Valid annotations
    └── response/    # Outbound payloads mapped via MapStruct
```

**Key decisions:**
- Entities use `UUID` PKs generated by the database (`GenerationType.UUID`)
- DTOs are mapped from/to entities via **MapStruct** — never expose entities directly in controllers
- **Lombok** is used on all entities and DTOs (`@Getter`, `@Setter`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`)
- Lombok and MapStruct annotation processors must both be declared in `maven-compiler-plugin`'s `annotationProcessorPaths` (already set in `pom.xml`) — order matters: Lombok before MapStruct
- `open-in-view: false` — services must load all needed associations before returning to the controller
- JWT secret and DB credentials come from env vars; defaults only exist for local dev

**Profiles:**
| Profile | DB | Flyway | DDL |
|---------|-----|--------|-----|
| `dev` (default) | PostgreSQL (localhost:5432) | enabled | validate |
| `prod` | PostgreSQL (env vars) | enabled | validate |
| `test` | H2 in-memory (PostgreSQL mode) | disabled | create-drop |

Flyway migrations go in `src/main/resources/db/migration/` (not yet created — add `V1__...sql` files as schema grows).

**Testcontainers** is available for tests that need real PostgreSQL behaviour. Use `docker-compose.test.yml` (tmpfs, port 5433) as the container target in those tests.

### Frontend

Angular 17 standalone-component application (no NgModules). Bootstrapped in `src/main/ts` via `appConfig`.

```
src/app/
├── app.config.ts    # root providers: Router, AnimationsAsync
├── app.routes.ts    # lazy-loaded route definitions
└── app.component.*  # root shell component
```

Angular Material (indigo-pink theme) is the UI component library. New feature modules should follow the pattern: `feature/feature-name/` with standalone components, a `feature-name.routes.ts`, and a `feature-name.service.ts`.

HTTP calls to the backend go to `/api/` — in dev this requires a proxy config (`proxy.conf.json`); in production the Nginx reverse proxy in `frontend/nginx.conf` forwards `/api/` to `http://backend:8080/api/`.

## Environment Setup

Copy `.env.example` to `.env` in the project root before running Docker Compose. The backend reads `SPRING_PROFILES_ACTIVE`, `DB_*`, and `JWT_SECRET` from the environment. For local dev without Docker, export those vars or rely on the defaults in `application-dev.yml`.
